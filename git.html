    Comandos Git

    Git-Init Git-Add Git-Status Usuario Git-Commit Git-Checkout Git-Diff Git-branch Git-Clone Git-Log Git-Push Git-Pull

    Git-Init


    Git-Init: Crea un repositorio de Git vacío o en otras palabras git init es para inicializar un proyecto

    Git-Add

    Git-Add: Agrega el contenido del archivo al índice. Este comando actualiza el índice utilizando el contenido actual que se encuentra en el árbol de trabajo, para preparar el contenido preparado para la próxima confirmación. Por lo general, agrega el contenido actual de las rutas existentes como un todo, pero con algunas opciones también se puede usar para agregar contenido con solo una parte de los cambios realizados en los archivos del árbol de trabajo aplicados, o eliminar las rutas que no existen en el árbol de trabajo.

    Git-Status


    Git-Status: Muestra el estado del árbol de trabajo, Muestra con que archivos se trabaja los que no están agregado están color rojo y cuando ya se agregaron se muestra de color verde  

    Usuario

    git config --global user.email y git config --global user.name. Configurar su nombre de usuario y direccion de correo electronico. Esto es importante porque cada confirmacion de Git usa esta informacion y esta integrada de forma inmutable en las confirmaciones que comienzas a crear.

    Git-Commit

    Git-Commit: Registra los cambios en el repositorio, Cree una nueva confirmación que contenga el contenido actual del índice y el mensaje de registro dado que describe los cambios.

    Git-Checkout

    Git-Checkout: Cambia de rama o restaura archivos de árbol de trabajo. Actualiza archivos en el árbol de trabajo para que coincidan con la versión en el índice o el árbol especificando.

    Git-Diff

    Git-Diff: Muestra los cambios entre confirmaciones, confirmaciones y árbol de trabajo, etc. Muestra cambios entre el árbol de trabajo y el índice o un árbol, cambios entre el índice y un árbol, cambios entre dos árboles, cambios resultantes de una fusión, cambios entre dos objetos blob o cambios entre dos archivos en el disco.

    Git-branch

    Git-branch: Enumera, crea o elimina ramas. Si --list se proporciona, o si no hay argumentos que no sean de opción, se enumeran las ramas existentes; la rama actual se resaltará en verde y se marcará con un asterisco. Todas las ramas registradas en árboles de trabajo vinculados se resaltarán en cian y se marcarán con un signo más. La opción -r hace que se enumeren las ramas de seguimiento remoto y la opción -a muestra las ramas locales y remotas.

    Git-Clone

    Git-Clone: Clona un repositorio en un directorio recién creado, crea ramas de seguimiento remoto para cada rama en el repositorio clonado (visible usando  git branch --remotes), y crea y extrae una rama inicial que se bifurca desde la rama actualmente activa del repositorio clonado.

    Git-Log

    Git-Log: Muestra los registros de confirmación. Enumere las confirmaciones a las que se puede acceder siguiendo los parent enlaces de las confirmaciones dadas, pero excluya las confirmaciones a las que se puede acceder de las dadas con un ^ delante de ellas. 

    Git-Push

    Git-Push: Actualiza referencias remotas usando referencias locales, mientras envía los objetos necesarios para completar las referencias dadas.

    Git-Pull

    Git-Pull Incorpora cambios de un repositorio remoto en la rama actual. En su modo predeterminado, git pulles la abreviatura de git fetchseguido de git merge FETCH_HEAD.
